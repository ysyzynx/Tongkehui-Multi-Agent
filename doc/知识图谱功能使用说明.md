# 童科绘知识图谱功能使用说明

## 功能概述

童科绘知识图谱功能为儿童科普内容创作提供了强大的知识关联和可视化探索能力。

## 技术架构

- **数据存储**: SQLite + JSON字段
- **图计算**: NetworkX (可选)
- **前端可视化**: AntV G6
- **实体提取**: LLM + 规则混合方式

## 核心功能

### 1. 图谱探索
- 可视化展示科学概念之间的关联
- 支持搜索和定位实体
- 点击节点查看详情和展开邻居
- 力导向图布局，直观展示关系网络

### 2. 实体管理
- 10种实体类型（概念、物体、生物、人物、星球、现象、过程、设备、地点、事件）
- 支持手动创建和编辑实体
- 向量嵌入支持语义搜索

### 3. 图谱构建
- **从文档提取**: 自动从现有知识库文档中提取实体和关系
- **维基百科集成**: 输入维基百科页面标题，一键提取知识
- **置信度评分**: 每个实体和关系都有置信度标记

### 4. 图计算（需安装NetworkX）
- 核心实体识别（PageRank算法）
- 社区发现（知识聚类）
- 路径查找（发现实体间的关联路径）

## 实体类型说明

| 类型 | 说明 | 示例 |
|------|------|------|
| CONCEPT | 科学概念 | 重力、光合作用、能量 |
| OBJECT | 物体/物质 | 水、恐龙、太阳 |
| ORGANISM | 生物 | 蜜蜂、榕树、蓝鲸 |
| PERSON | 人物 | 爱因斯坦、达尔文 |
| PLANET | 星球 | 地球、火星、月球 |
| PHENOMENON | 自然现象 | 彩虹、火山爆发 |
| PROCESS | 过程/原理 | 水循环、消化 |
| DEVICE | 设备/工具 | 显微镜、望远镜 |
| PLACE | 地点 | 亚马逊雨林、南极 |
| EVENT | 事件 | 恐龙灭绝、人类登月 |

## 关系类型说明

| 类型 | 说明 | 示例 |
|------|------|------|
| IS_A | 是一种 | 蜜蜂 IS_A 昆虫 |
| PART_OF | 是...的一部分 | 心脏 PART_OF 循环系统 |
| HAS_PART | 包含 | 太阳 HAS_PART 日冕 |
| CAUSES | 导致 | 温室效应 CAUSES 全球变暖 |
| IS_CAUSED_BY | 由...导致 | 地震 IS_CAUSED_BY 板块运动 |
| RELATED_TO | 与...相关 | 蜜蜂 RELATED_TO 花粉 |
| INTERACTS_WITH | 与...相互作用 | 植物 INTERACTS_WITH 阳光 |
| LIVES_IN | 生活在 | 企鹅 LIVES_IN 南极 |
| DISCOVERED_BY | 由...发现 | 万有引力 DISCOVERED_BY 牛顿 |
| EXAMPLE_OF | 是...的例子 | 恐龙 EXAMPLE_OF 古生物 |
| SIMILAR_TO | 类似于 | 蝙蝠 SIMILAR_TO 鸟类 |
| CONTRASTS_WITH | 与...形成对比 | 冬眠 CONTRASTS_WITH 夏眠 |

## 快速开始

### 1. 安装依赖

```bash
# 后端依赖
pip install networkx

# 前端依赖（在tk-frontend目录下）
cd tk-frontend
npm install @antv/g6
```

### 2. 数据库迁移

系统会自动在首次启动时创建知识图谱相关的数据库表。

### 3. 访问知识图谱页面

启动应用后，访问:
- 首页 → 知识库 → 点击"知识图谱"按钮
- 或直接访问 `/knowledge-graph` 路径

### 4. 构建第一个图谱

**方式一：从维基百科提取**
1. 进入"图谱构建"标签页
2. 输入维基百科页面标题（如"光合作用"）
3. 点击"开始提取"
4. 系统会自动提取实体和关系并展示在图谱中

**方式二：从现有知识库文档提取**
1. 确保知识库中已有文档
2. 使用API `/api/kg/extract-from-document` 从指定文档提取
3. 或使用 `/api/kg/build-from-knowledge-base` 批量处理

## API接口说明

### 实体管理

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/kg/entities` | GET | 获取实体列表 |
| `/api/kg/entities/{id}` | GET | 获取单个实体详情 |
| `/api/kg/entities` | POST | 创建新实体 |
| `/api/kg/entities/{id}` | PUT | 更新实体 |
| `/api/kg/entities/{id}` | DELETE | 删除实体 |

### 关系管理

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/kg/relations` | GET | 获取关系列表 |
| `/api/kg/relations/{id}` | GET | 获取单个关系详情 |
| `/api/kg/relations` | POST | 创建新关系 |
| `/api/kg/relations/{id}` | PUT | 更新关系 |
| `/api/kg/relations/{id}` | DELETE | 删除关系 |

### 图谱检索

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/kg/entities/search` | POST | 搜索实体（支持语义搜索） |
| `/api/kg/entities/{id}/neighbors` | GET | 获取实体的邻居节点 |
| `/api/kg/subgraph` | GET | 获取子图 |
| `/api/kg/path` | GET | 查找两个实体之间的路径 |

### 图谱构建

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/kg/extract-from-document` | POST | 从文档提取知识图谱 |
| `/api/kg/extract-from-wikipedia` | POST | 从维基百科提取知识图谱 |
| `/api/kg/build-from-knowledge-base` | POST | 从整个知识库构建图谱 |

### 图计算

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/kg/central-entities` | GET | 获取核心实体 |
| `/api/kg/communities` | GET | 获取社区发现结果 |

### 其他

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/kg/stats` | GET | 获取知识图谱统计 |
| `/api/kg/types` | GET | 获取实体类型和关系类型定义 |

## 前端组件说明

### KnowledgeGraphPage
主知识图谱页面，包含三个标签页：
- **图谱探索**: 可视化展示和交互
- **实体管理**: 实体列表和管理
- **图谱构建**: 从各种来源构建图谱

### kg-api.ts
知识图谱API客户端，提供所有API的TypeScript封装。

## 使用示例

### 示例1：从维基百科提取"恐龙"相关知识

```typescript
import { extractFromWikipedia } from '../lib/kg-api';

const result = await extractFromWikipedia('恐龙', 'zh', true);
console.log(`提取了 ${result.entities.length} 个实体`);
console.log(`提取了 ${result.relations.length} 个关系`);
```

### 示例2：搜索相关实体

```typescript
import { searchEntities } from '../lib/kg-api';

const entities = await searchEntities('光合作用', 'CONCEPT', 10);
entities.forEach(entity => {
  console.log(`${entity.name} (${entity.entity_type})`);
});
```

### 示例3：查看实体的邻居

```typescript
import { getEntityNeighbors } from '../lib/kg-api';

const subgraph = await getEntityNeighbors(entityId, undefined, 2, 30);
console.log(`找到 ${subgraph.nodes.length} 个节点`);
console.log(`找到 ${subgraph.edges.length} 条边`);
```

## 注意事项

1. **LLM提取准确率**: 实体和关系提取依赖于LLM，可能存在误差，建议人工审核
2. **置信度评分**: 低置信度(<0.7)的实体和关系需要特别关注
3. **性能优化**: 图谱数据量大时，建议使用子图展示，避免一次性加载所有节点
4. **NetworkX可选**: 图计算功能需要安装NetworkX，但基础功能不依赖它
